使用指南

Guide

给日常使用者看的操作指南：该去哪个页面、什么时候记录、哪些地方最容易填错。

A practical guide for daily colony work: where to go, what to record, and how to avoid messy data.

先看这里

MouseXGene 用来管理小鼠、笼子、课题、品系、繁殖、窝仔和基因型结果。 最重要的原则是：实验室真实发生了什么，就在对应页面及时记录什么。 例如换笼、结束小鼠、开始配繁、出生一窝、离乳，都应该走各自的流程，不要只写在备注里。

  1. 查或修改单只动物：去 Mice。
  2. 查一个笼子现在有什么鼠：去 Cages。
  3. 记录配繁组合：去 Breedings。
  4. 记录出生和离乳：去 Litters。
  5. 维护品系名称、位点和 PDF 说明：去 Strain Lines。

每天怎么用

  1. 先打开 Home，看有没有缺出生日期、缺笼位、待离乳、繁殖逾期等提醒。
  2. 有 pups 要处理时，进 Litters，优先看未离乳或接近离乳日期的记录。
  3. 有新出生的一窝，从 Create Litter from Breeding 找到对应配繁记录后创建 litter。
  4. 有配繁结束，进 breeding detail，点击 End Breeding and Move Breeders。
  5. 要找一只鼠，在 Mice 用 UID、ear tag、toe tag 搜索。
  6. 要移动鼠，先确认目标笼子，再用 Move Cage。

核心概念

对象 是什么意思 什么时候用
Project 课题或管理归属。 给小鼠归属项目，方便筛选、权限管理和导出。
Strain Line 品系名称和预期基因位点。 新建小鼠、离乳、录基因型前先确认品系。
Cage 实际笼位或笼号。 记录当前笼内有哪些鼠、做换笼、查繁殖笼。
Mouse 一只具体小鼠。 记录 UID、性别、生日、笼位、项目、父母、备注和基因型。
Breeding 一次配繁设置：一只 sire 加一只或多只 dams。 配繁开始时创建；配繁结束时关闭并移动 breeders。
Litter 某次出生的一窝 pups。 出生时创建；离乳时生成单只 Mouse 记录。
Genotype 实际基因型检测结果。 PCR/测序结果出来后填写，不要只凭品系名推测。

Mice

新建小鼠

  1. 进入 Mice → New Mouse。
  2. 先填共享信息：birth date、status、strain line、project，如果已经在笼里，也选 current cage。
  3. 下面表格每一行是一只小鼠，填写 Mouse UID、性别、ear tag、toe tag。
  4. 如果这批鼠来源明确，再填 Parentage；外购、历史补录或父母不清楚时选 No parentage recorded。

Parentage 怎么选

  • No parentage recorded：父母未知、不重要、或外购鼠时使用。
  • Use breeding cage parents：这批鼠来自某个配繁笼时使用。若该 breeding 有两只或多只母鼠，系统会记录全部 possible dams，不会假装知道具体是哪一只母鼠。
  • Select sire and possible dam(s)：手动选择父亲和可能母亲。确定母亲时选 1 只；不确定时可选多只。

编辑、换笼、结束小鼠

  • 一只小鼠从出生到死亡只使用一个 Mouse UID。状态、笼位、项目或备注可以变化，但 Mouse UID 不变，也不能复用给另一只鼠。
  • Edit Mouse：修改 UID 以外的基础信息、project、strain line、parentage、notes、status 等。
  • Move Cage：活鼠换笼时使用，会更新当前笼位和笼位历史。
  • End Mouse：小鼠安乐死或进入终态时使用。系统会弹窗确认，并把它从当前笼位占用中移除。
  • Edit Genotyping Results：只用于修改真实基因型结果，不是修改小鼠基础信息的入口。

Cages

  • 能先建笼子就先建笼子，再把小鼠放进去。
  • Room、Rack、Position 尽量用统一格式，否则以后筛选会很乱。
  • 一个真实笼位只使用一个 Cage ID。笼子状态、用途、房间位置可以变化，但 Cage ID 不变，也不能复用给另一个笼位。
  • Cage detail 里的 Current Mice 是当前占笼情况。鼠 UID 和 breeding 编号可以点击进入详情。
  • 普通情况下不要硬删除笼子；不用的笼子设为 inactive 或 retired 更安全。

Breedings

创建配繁

  1. 进入 Breedings → New Breeding。
  2. 选择 1 只 Sire，再选择 1 只或多只 Dams。
  3. 选择实际配繁所在的 breeding cage。
  4. 配繁开始时就创建 breeding，不要等到 pups 出生后才补。

如果确实需要，父母可以来自不同 project 或不同 owner。系统允许这样做，但会给出提示；保存前请确认这确实是实验设计需要。

从 breeding 创建 litter

  • 出生一窝后，使用 Create Litter from Breeding。
  • 如果 breeding 太多，可以按 breeding code、cage、project、strain line、owner、status 筛选。
  • birth date 尽量准确，因为后面的 weaning reminder 依赖这个日期。

结束 breeding

  • 配繁结束、父母分开时，使用 End Breeding and Move Breeders。
  • 每只 breeder 都要选择下一步去哪个笼子。
  • 如果目标笼不存在，在该页面点 Create New Cage，建好后回到原页面继续。
  • Exception: no current cage 只给真实例外使用，不要作为省略目标笼的常规做法。

Litters

Litter 是什么

Litter 是某一次出生的一窝 pups，不等于 breeding。 一个 breeding 可以在不同时间产生多窝 litters。

  • Active / not weaned：还需要处理，通常是日常工作重点。
  • Weaned：已经离乳，并生成了单只 Mouse 记录。
  • Ended / archived：流程已经关闭，通常只作为历史记录查看。

Wean Litter 怎么做

  1. 进入 litter detail，点击 Wean。
  2. 先确认 Parentage。如果是双母或多母配繁，且不知道具体母亲，使用 breeding cage parentage。
  3. 填写 male / female pup 数量，再逐只填 UID 和标签。
  4. 选择公鼠和母鼠的目标笼位。某一性别数量为 0 时，不需要填该性别笼位。
  5. 选择 pups 的 project：父方 project、母方 project，或创建一个新 project。
  6. 选择 pups 的 strain line 来源：父方 strain、母方 strain，或创建一个新 strain line。
  7. 提交前检查 UID 不重复、目标笼正确、project 权限正确。

Wean 成功后，系统会创建单只 Mouse 记录、写入笼位历史、记录谱系来源，并生成基因型模板行。 正常情况下 litter 会变成 Weaned 状态。

Strain Lines

  • Strain line 名称应尽量唯一、稳定。不要为了不同用户重复创建多个同名 WT。
  • Projects 表示这条品系可以属于或用于哪些 project。
  • Owners using this line 显示当前哪些 owner 的数据正在使用这条品系。
  • 维护 Included loci 后，新建小鼠和 wean 时才会自动出现更合理的 genotype rows。
  • PDF 可用于上传饲养说明、基因型 protocol、品系说明等文档。

WT 小鼠建议共用一条 WT strain line。不同用户、不同课题的 WT，用 Project、Owner、笼位、UID 和 notes 区分。

Genotypes

  • Genotype rows 主要来自小鼠 strain line；如果有父母信息，也会参考父母 strain line 的位点。
  • X-linked 和 Y-linked 位点会根据性别显示不同选项。
  • 下拉框没有真实结果时，选 Custom 后手动输入。
  • 实际 PCR 或测序结果出来后，从 mouse detail 进入 Edit Genotyping Results 修改。
  • 不要把 genotype 备注当成 project、strain line 或 pedigree 的替代字段。

列表、搜索、导出

  • 搜索框常用于 UID、cage ID、breeding code、ear tag、toe tag 等可见标识。
  • 编辑或导出前先筛选，减少误操作。
  • 如果页面看起来“少了很多数据”，先点 Reset，可能是旧筛选条件还在。
  • 日常使用分页即可；只有结果很少时再用 All。
  • 需要分析、备份或发给别人时，用 export，而不是从网页手工复制。

Import

日常新增少量小鼠，推荐直接用页面表单录入，不建议频繁用 Excel import。 Import 更适合历史数据清理、一次性大量录入或管理员维护。

  • 导入前先下载系统模板，不要改列名。
  • Mouse UID 和 cage ID 必须唯一。
  • 日期统一使用 YYYY-MM-DD。
  • status、sex、cage use 等字段要使用系统允许的值。
  • 如果导入自动创建了 project、strain line 或 cage，导入后要回头检查这些占位数据是否合理。
  • 不确定时，先找 manager 或 admin 确认再导入。

权限

  • Admin：可以管理用户、角色、审计日志和全局数据。
  • Manager：通常可以维护 project 数据和执行常见工作流。
  • Member：可以查看和使用常规页面，但可能不能导入、归档或管理成员。
  • 如果系统阻止保存，先确认你是否在正确 project 下，以及账号是否有对应权限。

好习惯

  • 换笼当天就记录 Move Cage。
  • 不要重复使用 mouse UID 或 cage ID。鼠和笼子的状态可以变，编号不变。
  • notes 用来补充背景，结构化信息尽量填在对应字段里。
  • breeding 结束时及时关闭并移动 breeders。
  • 查看 litters 时，把 weaned 和 not weaned 分开看。
  • 普通历史记录优先使用 inactive、retired、archived 或终态，不要随意删除。

常见问题

为什么一只 pup 可以有多个 possible dams？
双母或多母配繁时，常常不知道具体是哪只母鼠生的。系统记录全部可能母亲，是为了让谱系更诚实。
Wean 后 litter 要不要自动 end？
Weaned 表示已经离乳并生成单只鼠；ended / archived 表示记录完全关闭。两者可以分开，除非实验室决定所有 weaned litters 都立即关闭。
为什么 End Breeding 要选 destination cage？
配繁结束后 breeders 会离开 breeding cage。系统需要知道它们去了哪里，才能保持当前笼位占用准确。
Picker 里找不到鼠或笼子怎么办？
先检查 owner、project、strain line、include inactive 等筛选条件。必要时点 Reset 或换关键词搜索。
录错了怎么办？
普通错误直接 edit；涉及历史数据、删除、大量修改时，先找 admin 确认。
页面很慢或结果不对怎么办？
先清空筛选、缩小结果、硬刷新浏览器。仍然不对时，把具体 URL 和你要做的操作告诉管理员。

Start Here

MouseXGene tracks mice, cages, projects, strain lines, breeding setups, litters, and genotype results. The most important rule is: record the real lab event in the workflow that matches it. Cage moves, terminal mouse status, new breedings, litter births, and weaning should each be recorded as structured data, not only in notes.

  1. Find or edit one animal in Mice.
  2. Check current cage occupancy in Cages.
  3. Record mating setups in Breedings.
  4. Record births and weaning in Litters.
  5. Maintain line names, loci, and PDF documents in Strain Lines.

Daily Checklist

  1. Open Home first and scan alerts for missing birth dates, missing cages, pending weaning, or overdue breedings.
  2. When pups need attention, open Litters and focus on not-weaned or near-weaning records.
  3. When a litter is born, use Create Litter from Breeding to find the matching breeding and create the litter.
  4. When a breeding ends, open the breeding detail page and click End Breeding and Move Breeders.
  5. To find one mouse, search in Mice by UID, ear tag, or toe tag.
  6. Before moving a mouse, confirm the destination cage, then use Move Cage.

Core Concepts

Item Meaning When to use it
Project A research project or management owner. Assign mice to projects for filtering, permissions, and export.
Strain Line The line name and expected genetic loci. Confirm it before creating mice, weaning pups, or entering genotypes.
Cage A real cage or cage position. Track current occupancy, cage moves, and breeding cages.
Mouse One individual animal. Record UID, sex, birth date, cage, project, parents, notes, and genotypes.
Breeding One mating setup: one sire plus one or more dams. Create it when mating starts; close it and move breeders when it ends.
Litter One birth event from a breeding. Create it at birth; wean it to generate individual mouse records.
Genotype The actual genotyping result. Enter it after PCR or sequencing results are available. Do not infer it from the strain name alone.

Mice

Create mice

  1. Go to Mice → New Mouse.
  2. Fill shared information first: birth date, status, strain line, project, and current cage if known.
  3. Each row in the table is one mouse. Fill Mouse UID, sex, ear tag, and toe tag.
  4. Use Parentage only when the source is known. For external, historical, or unknown animals, choose No parentage recorded.

How to choose Parentage

  • No parentage recorded: use this when parents are unknown, irrelevant, or external.
  • Use breeding cage parents: use this when the mice came from a breeding cage. If the breeding has two or more dams, the system records all possible dams instead of pretending one exact dam is known.
  • Select sire and possible dam(s): manually choose the sire and possible dam(s). Select one dam if exact, or multiple dams if uncertain.

Edit, move, or end a mouse

  • Each mouse keeps one Mouse UID from birth to death. Status, cage, project, and notes may change, but the Mouse UID stays the same and must not be reused for another mouse.
  • Edit Mouse: edit basic information, project, strain line, parentage, notes, status, and similar fields.
  • Move Cage: use this when a live mouse changes cage. It updates current cage and cage history.
  • End Mouse: use this when a mouse is euthanized or reaches a terminal status. The system asks for confirmation and removes the mouse from current cage occupancy.
  • Edit Genotyping Results: use this only for actual genotype calls, not for basic mouse information.

Cages

  • Create the cage before assigning mice whenever possible.
  • Keep Room, Rack, and Position formats consistent, otherwise filtering becomes messy.
  • Each real cage position keeps one Cage ID. Status, cage use, room, rack, and position may change, but the Cage ID stays the same and must not be reused for another cage position.
  • Current Mice on cage detail is the live occupancy view. Mouse UIDs and breeding codes are clickable.
  • Do not hard-delete normal cages. Use inactive or retired status when a cage is no longer in routine use.

Breedings

Create a breeding

  1. Go to Breedings → New Breeding.
  2. Select one Sire and one or more Dams.
  3. Select the actual breeding cage.
  4. Create the breeding when mating starts, not after pups are born.

Parents can come from different projects or owners if the experimental design requires it. The system allows this and shows warnings so you can confirm before saving.

Create litter from breeding

  • After pups are born, use Create Litter from Breeding.
  • If there are many breedings, filter by breeding code, cage, project, strain line, owner, or status.
  • Enter birth date as accurately as possible, because weaning reminders depend on it.

End breeding

  • When the breeding setup is finished and parents are separated, use End Breeding and Move Breeders.
  • Choose the next cage for each breeder.
  • If the destination cage does not exist, click Create New Cage on that page, then return and finish.
  • Use Exception: no current cage only for real exceptions, not as the normal way to skip a destination cage.

Litters

What a litter means

A litter is one birth event from a breeding. It is not the same as a breeding. One breeding can produce multiple litters over time.

  • Active / not weaned: still needs attention and is usually the daily priority.
  • Weaned: pups have been weaned and individual Mouse records have been created.
  • Ended / archived: the workflow is closed and usually kept for history only.

How to wean a litter

  1. Open litter detail and click Wean.
  2. Confirm Parentage. If the breeding has multiple dams and the exact dam is unknown, use breeding cage parentage.
  3. Enter male and female pup counts, then fill each pup UID and tags.
  4. Select destination cages for male and female pups. A sex with count 0 does not need a cage.
  5. Choose the pups' project: sire project, dam project, or a new project.
  6. Choose the pups' strain line source: sire strain, dam strain, or a new strain line.
  7. Before submitting, check UID uniqueness, destination cages, and project permissions.

After weaning, the system creates individual Mouse records, cage history, pedigree links, and genotype template rows. Normally the litter becomes Weaned.

Strain Lines

  • Strain line names should be unique and stable. Do not create duplicate WT strain lines for different users.
  • Projects shows which projects this line belongs to or can be used in.
  • Owners using this line shows which owners currently have data using this line.
  • Maintain Included loci so new mouse and weaning forms can show useful genotype rows.
  • Use PDFs for husbandry notes, genotyping protocols, and line documentation.

WT mice should usually share one WT strain line. Separate different users' WT colonies with Project, Owner, cage, UID, and notes.

Genotypes

  • Genotype rows mainly come from the mouse strain line. When parentage exists, parent strain lines can also contribute loci.
  • X-linked and Y-linked loci show different options based on mouse sex.
  • If the dropdown does not contain the real result, choose Custom and type it manually.
  • After PCR or sequencing results are available, open mouse detail and use Edit Genotyping Results.
  • Do not use genotype notes as a replacement for project, strain line, or pedigree fields.

Lists, Search, and Export

  • Search commonly works best with UID, cage ID, breeding code, ear tag, toe tag, and other visible identifiers.
  • Filter before editing or exporting to reduce accidental work on the wrong records.
  • If a page looks like many records are missing, click Reset; an old filter may still be active.
  • Use pagination for daily work. Use All only for small result sets.
  • Use export for analysis, backup, or sharing a snapshot. Avoid copying rows manually from the page.

Import

For small daily additions, use the regular forms instead of Excel import. Import is better for historical cleanup, one-time large entry, or administrator maintenance.

  • Download the system template first and do not rename columns.
  • Mouse UID and cage ID must be unique.
  • Use YYYY-MM-DD dates.
  • Use allowed system values for status, sex, and cage use.
  • If import creates placeholder projects, strain lines, or cages, review them afterward.
  • When unsure, ask a manager or admin before importing.

Permissions

  • Admin: can manage users, roles, audit logs, and global data.
  • Manager: can usually maintain project data and run common workflows.
  • Member: can view and use routine pages, but may not be able to import, archive, or manage members.
  • If the system blocks saving, first check whether you are in the correct project and whether your account has the required role.

Good Habits

  • Record Move Cage on the day a cage move happens.
  • Do not reuse mouse UIDs or cage IDs. Status can change; identifiers stay the same.
  • Use notes for background context, but keep structured information in structured fields.
  • Close breedings and move breeders when breeding ends.
  • Review weaned and not-weaned litters separately.
  • For normal historical records, prefer inactive, retired, archived, or terminal status instead of deleting.

Common Questions

Why can one pup have multiple possible dams?
In two-dam or multi-dam breeding, the exact mother is often unknown. Recording all possible dams keeps the pedigree honest.
Should weaning automatically end a litter?
Weaned means individual mouse records have been created. Ended or archived means the record is fully closed. Keep them separate unless the lab decides all weaned litters should close immediately.
Why does End Breeding require destination cages?
When breeding ends, breeders leave the breeding cage. The system needs their next cage to keep current occupancy correct.
What if a picker cannot find a mouse or cage?
Check owner, project, strain line, include inactive, and other filters first. Use Reset or a different search term if needed.
What if I entered something incorrectly?
Edit normal mistakes directly. For historical data, deletion, or bulk changes, ask an admin first.
What if a page is slow or results look wrong?
Clear filters, reduce the result set, and hard-refresh the browser. If it still looks wrong, tell the admin the exact URL and what you were trying to do.
