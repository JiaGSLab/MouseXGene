MouseXGene 使用指南（中文）

A. 系统概览

MouseXGene 用于实验室日常协作管理，覆盖 Projects、Mice、Cages、 Breedings、Litters、Strain Lines、 Genotypes 等核心对象。系统强调数据可追溯、状态可管理（active / archived / euthanized / retired）、 以及 import / export 的批量处理能力。

B. 推荐使用流程

  1. 先确认或新建 Strain Line，保证后续 Mouse 和 Genotype 记录有统一命名基础。
  2. 新建 Cage；如果使用 import Mouse 且开启自动创建，也可在导入时自动补齐 Cage。
  3. 新建或 import Mouse，优先保证核心身份字段完整。
  4. 补录或 import Genotype，完善每只 Mouse 的遗传信息。
  5. 记录 Breeding 和 Litter，形成连续繁殖与后代追踪链路。

C. Import 说明

  • 支持 import Cage、import Mouse、import Genotype。
  • Mouse 导入支持 mouse-centered import with genotype slots（单文件导入 Mouse + 分组基因型列）。
  • 导入时可按选项自动创建依赖对象：Strain Line、Project、Cage。
  • 常见失败原因：必填列缺失、日期格式不正确、重复主键、引用对象无法解析、字段取值超出允许范围。
  • 涉及 sire / dam 时，父母需要已存在，或与当前 Mouse 在同一个导入文件中可被解析。

D. 权限说明

系统采用基于角色（role）的权限控制。不同角色在“查看、编辑、导入、归档、项目管理、用户管理、审计查看”等方面拥有不同权限。下表用于快速说明各角色的主要能力范围。

功能 Admin Manager Member
查看全部业务数据（Mice / Cages / Breedings / Litters / Genotypes / Strain Lines） 是是是
新建普通业务记录 是是是
编辑普通业务记录 是是是
导入数据（import） 是是否
导出数据（export） 是是是
归档 / 停用记录（archive / deactivate） 是是否
新建 Project 是是否
编辑 Project 是是否
管理 Project 成员 是是否
查看 Audit 是否否
管理全局 Users / roles 是否否
进入 Django Admin 是否否

说明：

  1. “普通业务记录”通常包括 Mouse、Cage、Breeding、Litter、Genotype 等实验室日常数据。
  2. Manager 和 Member 均可参与日常协作录入，但高风险操作（如 import、archive、权限管理、Audit 查看）会受到限制。
  3. Admin 负责系统级管理；Manager 负责业务管理；Member 主要负责日常使用与记录维护。

E. 状态说明

  • active：正常在用状态，默认优先展示。
  • inactive：非活跃状态，通常表示暂不使用或待观察。
  • archived：已归档，保留历史记录但不作为日常工作重点。
  • euthanized：Mouse 已执行安乐，记录仍保留用于追溯。
  • retired：Cage 或其他对象已退役，不再用于当前流程。

列表中灰色显示通常表示对象处于 inactive / archived / retired 等非活跃状态。

F. 常见问题

为什么有些对象不会被真正删除？
系统采用历史保留策略，优先使用 archive / deactivate，避免影响审计和追溯链路。
为什么 sire / dam 需要先存在或在同一个 import 文件中？
因为父母关系需要在导入流程中解析到具体 Mouse，系统才能建立正确谱系。
为什么 Strain Line / Project / Cage 可能会自动创建？
当导入选项开启自动创建时，系统会补齐缺失依赖，减少手工前置维护。
为什么有些记录会显示为灰色？
灰色通常表示记录不是 active 状态（例如 archived / inactive / retired），用于提醒非日常主流程对象。
