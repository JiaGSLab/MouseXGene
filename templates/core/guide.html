Guide

面向一线使用者的 MouseXGene 实操指南：今天该点哪个页面、先填什么、为什么要这样做。

1. 先理解：你每天在系统里做什么

把它想成一条实验链路：先有 Project（属于哪个课题）、再有 Strain Line（品系定义）、 再到 Cage（笼位）、Mouse（个体）、Genotype（基因型）、 最后是 Breeding 与 Litter（繁殖过程与后代批次）。 系统的核心价值是：能追溯、能筛选、能导出，并把每次关键修改记录在履历里。

2. 首页（Dashboard）怎么用

  • 先看统计：Total/Active Mice 与 Cages 用于确认库存与在用状态。
  • 再看告警：缺基因型、缺出生日期、繁殖流程异常等，优先补齐“会影响后续判断”的数据。
  • 最后执行动作：Quick Actions 进入新建、导入、导出；“More actions”放低频操作。

3. 列表页通用操作（Mice / Cages / 等）

  • 先筛选再操作：先缩小范围（q、status、project 等），再编辑/导出，避免误改。
  • More filters：放次要条件；主筛选区放高频条件，降低误点。
  • 分页：Mice/Cages 支持每页 25/50/100；结果 ≤500 时可选 All。
  • 可点击列：链接一般表示“下钻查看详情”或“带筛选跳转到相关列表”。
  • 导出建议：批量分析或做备份时用导出，单条核对优先在详情页看最新状态。

4. 重点：Breedings 页面到底做什么（交配计划层）

Breeding 是“一个交配设置”的记录，不是“已经出生的后代”。 你可以把它理解为：计划与执行中的配对单。后续是否产仔、产了几窝，都会挂在这个配对单下面。

  • 什么时候建 Breeding：当你确认了 male / female（或双母）组合，并开始配对时就建。
  • 核心字段意义：`Breeding code` 是唯一标识；`Cage` 记录配对所在笼；`start_date / plug_date / expected_birth_date` 用于时间管理与提醒。
  • 状态怎么理解：常见是 active/planned/plugged/closed（具体以系统枚举为准）；关闭后通常表示该配对已结束，不再产生新的 workflow 主动作。
  • 页面怎么用：列表页用于看“当前有哪些配对在跑”；详情页用于核对配对信息并进入该 breeding 下的 litter 操作。
  • 常见错误：把“每一窝”当成 Breeding 新建；正确做法是同一对持续配对下，按实际分娩创建多个 litter。

5. 重点：Litters 页面到底做什么（后代批次层）

Litter 是“某次分娩产生的一批后代”的记录。 它是 Breeding 的下游对象，关注的是：这窝什么时候生、多少只、什么时候离乳、后续是否完成尾标/分拣。

  • 什么时候建 Litter：确认有一窝出生时，从对应 breeding 建 litter（而不是反过来）。
  • 核心字段意义：`birth_date` 决定后续离乳时间管理；`litter_size / males / females` 支持资源评估；`wean_date` 标记是否完成离乳节点。
  • 状态的业务意义：active/tailed/weaned/ended/archived（以系统枚举为准）反映流程节点，便于筛选“还要做什么”。
  • 页面怎么用：Litters 列表是“待办看板”——先筛 `not weaned` 看近期任务，再按日期范围或 breeding 缩小到当天工作范围。
  • 与 Mice 的关系：Litter 是“批次层”，Mouse 是“个体层”；通常先把批次流程跑通，再逐只完善个体记录。
  • Wean 时要决定 pups 归属项目：支持三种方式——归父方 project、归母方 project、或新建 project（详见下一节）。

6. Wean Litter 怎么处理（重点）

在 Litter detail → Wean 页面，系统会创建这窝 pups 对应的 Mouse 记录，并写入 cage membership。 你需要先确认目标笼位，再决定 pups 的项目归属。

  • Use sire project：pups 全部归父方项目；适合课题归属明确以父系线为主的场景。
  • Use dam project：pups 全部归母方项目；适合母系承担主课题或实际饲养归属在母方组的场景。
  • Create a new project：为这批后代建一个新项目；常用于跨项目配对后独立跟踪下一代。
  • 新建项目的 owner：默认是当前操作用户；系统会给该用户该项目的 manager membership，保证后续可编辑。
  • 权限校验：你选哪个项目，系统就校验你是否有该项目编辑权限；无权限会直接阻止提交。
  • 不再要求父母同项目：父方与母方来自不同项目也可以 wean，关键在于你给 pups 选择了合法的目标项目。

7. Wean 真实示例（跨项目配对）

例子：父鼠在 project1，母鼠在 project2，现在要给这一窝 pups 做 wean。 进入 Litter detail → Wean 后，可按实际管理需求选择：

  • 归父方项目：选 Use sire project (project1)，新生鼠全部写入 project1。
  • 归母方项目：选 Use dam project (project2)，新生鼠全部写入 project2。
  • 独立新项目：选 Create a new project，填写如 project3，新生鼠全部写入 project3。
  • 怎么选更稳妥：若后续由某一课题组持续维护，优先归该组项目；若是联合课题并准备独立追踪后代，优先新建项目。
  • 提交前检查：确认目标笼位、pup 数量、UID 不重复、目标项目你有编辑权限。

7. Breeding + Litter 的推荐操作流程（用户视角）

  1. 在 Breedings 创建/确认配对，检查配对笼与父母个体是否正确。
  2. 出现分娩后，在该 breeding 下创建 Litter，立即补 `birth_date`。
  3. 日常在 Litters 页面按 `not weaned` 与日期筛选，推进离乳、尾标、分拣节点。
  4. 进入 Wean 页面时，明确 pups 要归到父方/母方/新建项目，再批量生成个体鼠记录。
  5. 离乳后检查 litter 状态，并在需要时进入 Mice 逐只补全基因型与标签信息。
  6. 配对结束后回到 Breedings 关闭该 breeding，保持流程闭环。

9. Strain Lines

品系行是基因命名与统计的「锚点」。列表顶部为单行搜索 + 状态下拉 + 应用/重置，布局与 Mice/Cages 的筛选条一致。 行内数字可链到对应小鼠、笼位、交配或产仔列表，用于从品系下钻到个体。

10. Pedigree（谱系）

从小鼠详情进入 View Pedigree 后，页面以树状结构展示：上方为父母节点，经竖线汇聚到中间的焦点个体，下方分支列出同窝与后代，并用横线模拟树枝连接，便于快速扫读关系。 所有 UID 仍可点击进入对应小鼠详情。

11. 推荐录入顺序（新课题）

  1. 确认或新建 Strain Line，保证命名与统计口径统一。
  2. 新建 Project 并指定 Owner；后续小鼠、导出与协作范围常与此挂钩。
  3. 新建 Cage；若使用导入且开启自动创建，也可在导入小鼠时补笼。
  4. 新建或导入 Mouse，优先保证 UID、性别、品系、笼位、课题等核心字段。
  5. 补录或导入 Genotype，完善遗传信息。
  6. 按实验记录 Breeding / Litter，形成繁殖与后代链路。

12. Import / Export

  • 支持导入 Cage、Mouse、Genotype；小鼠导入可采用「单文件含基因型列」的格式。
  • 导入选项可自动创建缺失的 Strain Line、Project、Cage，减少手工前置维护。
  • 常见失败原因：必填列缺失、日期格式错误、主键重复、引用无法解析、枚举值越界。
  • 父母（sire/dam）须已存在，或与当前小鼠出现在同一导入文件中以便解析。
  • 导出：Mice/Cages 导出已按导入字段顺序优先输出，并可附带扩展列（owner、summary、timestamps 等）。

13. 权限说明（摘要）

系统按角色控制「谁能看、谁能改、谁能导入/归档、谁能管课题成员、谁能看审计」等。 下表为常见能力对比；若与贵实验室策略不一致，以管理员配置为准。

功能 Admin Manager Member
查看业务数据（Mice / Cages / Breedings / Litters / Genotypes / Strain Lines 等） 是是是
新建 / 编辑常规业务记录 是是是
导入（import） 是是否
导出（export） 是是是
归档 / 停用（archive / deactivate 等终态） 是是否
新建 / 编辑 Project，管理课题成员 是是否
查看 Audit（审计日志） 是否否
管理全局用户与角色、进入 Django Admin 是否否

补充：跨项目 breeding 时，操作人需要对涉及的每个项目都具备编辑权限；项目成员添加通常由该项目 owner、项目 manager 或 lab admin 执行。

14. 状态与列表中的灰色行

  • active：日常默认展示的主状态。
  • inactive：非活跃，可能暂停使用或待处理。
  • archived：归档保留历史，通常不作为当前工作重点。
  • euthanized：小鼠已安乐，记录保留用于追溯。
  • retired 等：笼位或其它对象已退役。列表中灰色行多表示非活跃/归档类状态。

15. 常见问题

为什么很少「硬删除」记录？
优先使用归档或终态标记，避免破坏审计与谱系链条。
导入时为什么父母找不到？
系统要把 sire/dam 解析成已存在的小鼠记录；请预先录入或放在同一导入文件中解析。
为什么会自动创建 Strain Line / Project / Cage？
开启对应导入选项时，会为缺失依赖自动建占位对象，减少打断导入流程。
列表里有些行发灰是什么意思？
多表示非活跃或归档状态，提醒不要与当前主流程混淆。
Project 后面的人名是谁？
一般为课题 Owner 的显示名（或全名/用户名），用于识别负责人。

16. 新手速查卡（30 秒）

  1. 先看 Dashboard 告警，处理“缺关键字段”的数据。
  2. 建新繁殖：进 Breedings，先建配对（不是先建 litter）。
  3. 有分娩：进该 breeding 下建 Litter，先填 `birth_date`。
  4. 每天任务：在 Litters 按 `not weaned` + 日期筛选推进流程。
  5. Wean 前先决定 pups 归属项目（父方/母方/新建），再提交创建。
  6. 查个体：在 Mice 用 UID 搜索，必要时进 pedigree 看家系。
  7. 批量分析或备份：优先用导出，避免在列表里手工逐条复制。
  8. 看不懂当前是谁登录：点顶栏 User: xxx 进入个人页确认账号。

17. 常见报错 FAQ（原因与处理）

导入时报 “Missing required columns”
文件缺少必需列或列名拼写不一致。请先下载系统模板，对齐列名后再导入。
导入时报 “invalid date ... Use YYYY-MM-DD format”
日期格式不符合要求。把 Excel/CSV 日期统一成 YYYY-MM-DD（例如 2026-04-20）。
导入时报 “duplicate ... in uploaded file”
同一个文件里出现重复主键（如重复 cage_id / mouse_uid）。去重后再导入。
导入时报 “... already exists in database”
数据库已有同名主键。可换新 ID，或按业务先归档旧记录再调整导入策略。
导入时报 “invalid status / invalid cage_type / invalid purpose”
枚举值超出系统允许范围。请使用下拉页面里可见的标准值，不要自定义拼写。
保存时报权限错误（例如不能编辑/归档）
当前账号权限不足。请联系 Admin/Manager，或用有权限账号执行该操作。
Wean 时提示没有项目编辑权限
你选择了某个 pups 归属项目，但当前账号对该项目无编辑权限。可改选你有权限的项目，或请项目 owner/manager/admin 先把你加入该项目。
Wean 选择“Create a new project”后提示项目名重复
项目名是全局唯一。请换一个新名称，或改选已有的父方/母方项目归属。
页面上显示 “No audit entries for this record yet.”
该记录目前没有可展示的审计条目（可能是老数据或仅有批量汇总日志）。不是系统故障。
导出后再导入失败
请确认你导入的是“包含导入必需列”的文件；扩展列可以有，但必需列不能删或改名。
为什么筛选结果和预期不一致？
先点 Reset 清空旧筛选，再逐个条件重新勾选；特别注意 `include inactive` 与分页设置。
为什么改了数据但页面像没更新？
先刷新并清缓存；若是服务器部署后未更新，优先检查代码同步、容器重建与 collectstatic 是否成功。
