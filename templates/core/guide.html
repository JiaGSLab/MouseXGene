Guide

MouseXGene 使用指南（中文）：面向日常进笼、繁殖与数据维护的实操说明。

1. 系统能做什么

MouseXGene 把实验室里常用的对象串成一条可追溯的链路： Projects（课题与归属）、Strain Lines（品系命名）、Cages（笼位与位置）、 Mice（个体身份与状态）、Genotypes（基因型记录）、 Breedings 与 Litters（交配设置与产仔/离乳流程）。 列表页支持搜索与筛选；多数对象支持 CSV / Excel 导出；有权限的用户可使用 批量导入。

2. 首页（Dashboard）

  • 统计卡片：总笼位数、活跃笼位、总小鼠数、活跃小鼠数；图标分别对应笼架与小鼠，便于一眼区分。
  • 数据质量提示：例如缺基因型摘要、缺出生日期等，用于提醒补录。
  • Quick Actions：新建小鼠、笼位、（有权限时）新建交配、导入与导出入口集中在「更多」折叠区，避免首屏过于拥挤。

3. 列表页通用操作（Mice / Cages / 等）

  • 搜索与筛选：顶栏为常用条件；「More filters」里是次要条件（例如是否包含非活跃记录）。字号与主筛选区一致，避免误点。
  • Mice / Cages 列表：筛选区底部可选择每页条数（25 / 50 / 100），变更后会自动刷新；当筛选结果总数 ≤500 时可选全部一次展示；超过 500 条时「全部」不可用并会提示改用筛选或导出。表格下方有分页链接（多于 1 页时显示）。
  • Mice 列表：Mouse UID 为进入个体详情页的链接；Age 列标题可点击，在「最年长优先 → 最年幼优先 → 恢复默认排序」之间循环；无出生日期的个体会排在同模式下的末尾。
  • Project / Owner：Project 与 Owner（课题负责人显示名）为两列，便于区分同名课题。
  • 年龄提示：缺生日会标「Age unknown」；高龄阈值（如 ≥6 月 / ≥1 年）在表头说明中有解释。
  • More actions：导出、基因型相关入口等放在工具栏折叠区，与主按钮区分层级。

4. Breedings 与 Litters

  • Breedings：描述一对（或多雌）交配设置、笼位、状态与关键日期；搜索框与下拉框样式与 Mice/Cages 列表对齐，便于跨页切换时视觉一致。
  • Litters：对应一次分娩批次；可记录离乳、尾标、与 workflow 状态。日期范围、是否已离乳等筛选项与主列表控件高度、字号一致。
  • 两页之间可用页头按钮互相跳转；具体编辑权限以服务器端校验为准。

5. Strain Lines

品系行是基因命名与统计的「锚点」。列表顶部为单行搜索 + 状态下拉 + 应用/重置，布局与 Mice/Cages 的筛选条一致。 行内数字可链到对应小鼠、笼位、交配或产仔列表，用于从品系下钻到个体。

6. Pedigree（谱系）

从小鼠详情进入 View Pedigree 后，页面以树状结构展示：上方为父母节点，经竖线汇聚到中间的焦点个体，下方分支列出同窝与后代，并用横线模拟树枝连接，便于快速扫读关系。 所有 UID 仍可点击进入对应小鼠详情。

7. 推荐录入顺序（新课题）

  1. 确认或新建 Strain Line，保证命名与统计口径统一。
  2. 新建 Project 并指定 Owner；后续小鼠、导出与协作范围常与此挂钩。
  3. 新建 Cage；若使用导入且开启自动创建，也可在导入小鼠时补笼。
  4. 新建或导入 Mouse，优先保证 UID、性别、品系、笼位、课题等核心字段。
  5. 补录或导入 Genotype，完善遗传信息。
  6. 按实验记录 Breeding / Litter，形成繁殖与后代链路。

8. Import / Export

  • 支持导入 Cage、Mouse、Genotype；小鼠导入可采用「单文件含基因型列」的格式。
  • 导入选项可自动创建缺失的 Strain Line、Project、Cage，减少手工前置维护。
  • 常见失败原因：必填列缺失、日期格式错误、主键重复、引用无法解析、枚举值越界。
  • 父母（sire/dam）须已存在，或与当前小鼠出现在同一导入文件中以便解析。
  • 导出：在对应列表或首页「更多」中使用 CSV/Excel；导出范围仍受权限与数据范围约束。

9. 权限说明（摘要）

系统按角色控制「谁能看、谁能改、谁能导入/归档、谁能管课题成员、谁能看审计」等。 下表为常见能力对比；若与贵实验室策略不一致，以管理员配置为准。

功能 Admin Manager Member
查看业务数据（Mice / Cages / Breedings / Litters / Genotypes / Strain Lines 等） 是是是
新建 / 编辑常规业务记录 是是是
导入（import） 是是否
导出（export） 是是是
归档 / 停用（archive / deactivate 等终态） 是是否
新建 / 编辑 Project，管理课题成员 是是否
查看 Audit（审计日志） 是否否
管理全局用户与角色、进入 Django Admin 是否否

10. 状态与列表中的灰色行

  • active：日常默认展示的主状态。
  • inactive：非活跃，可能暂停使用或待处理。
  • archived：归档保留历史，通常不作为当前工作重点。
  • euthanized：小鼠已安乐，记录保留用于追溯。
  • retired 等：笼位或其它对象已退役。列表中灰色行多表示非活跃/归档类状态。

11. 常见问题

为什么很少「硬删除」记录？
优先使用归档或终态标记，避免破坏审计与谱系链条。
导入时为什么父母找不到？
系统要把 sire/dam 解析成已存在的小鼠记录；请预先录入或放在同一导入文件中解析。
为什么会自动创建 Strain Line / Project / Cage？
开启对应导入选项时，会为缺失依赖自动建占位对象，减少打断导入流程。
列表里有些行发灰是什么意思？
多表示非活跃或归档状态，提醒不要与当前主流程混淆。
Project 后面的人名是谁？
一般为课题 Owner 的显示名（或全名/用户名），用于识别负责人。
